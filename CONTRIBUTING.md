# Contributing

OpenForge Core welcomes beginner-friendly contributions. Start with documentation, examples, templates, validation, and small runtime improvements before changing broad architecture.

## Development Flow

1. Create a focused branch.
2. Keep changes scoped to one feature or fix.
3. Update docs alongside code.
4. Run validation before opening a pull request.
5. Use clear commit messages.

## Pull Request Expectations

- Describe the user-facing or developer-facing change.
- Link related issues.
- Include validation evidence.
- Call out risks and follow-up work.
